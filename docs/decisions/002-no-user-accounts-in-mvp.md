# Architecture Decision Record 002: No User Accounts in MVP

**状态**：已批准  
**日期**：2026-07-01  
**决策人**：Solo Founder + AI Assistant

---

## 上下文

长期产品规划包括用户账号、宠物档案、订阅、营养计划等。但在 MVP 阶段，需要决定是否引入账号系统。

## 考虑的选项

| 选项 | 优点 | 缺点 |
|------|------|------|
| MVP 包含账号 | 可收集用户数据、早期测试个性化 | 显著增加开发复杂度、延迟上线、降低首访体验 |
| MVP 不包含账号 | 快速上线、专注内容、SEO 友好 | 无法个性化、无法留存用户 |

## 决策

MVP **不包含用户账号**。所有内容公开访问，无需登录。

## 理由

1. MVP 的核心假设是：宠物主人会通过搜索找到内容。
2. 账号系统会分散资源，延迟内容覆盖和 SEO 效果。
3. 没有账号，用户可以更快获得答案，降低跳出率。
4. 用户数据收集可以在验证需求后（Phase 2）再引入。

## 影响

- 食物详情页、搜索、分类页全部公开。
- 不使用数据库，不需要认证服务。
- 分析仅使用匿名数据。

## 后续阶段

Phase 2 引入用户账号和宠物档案，为个性化和订阅做准备。
